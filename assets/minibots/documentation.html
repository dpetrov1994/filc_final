A) Introduction, what are bots - top

Bots are small spiders/crawlers that goes out on internet and retrieves informations and data. With bots you can make very useful things, but bots and spider should be used carefully, because a bad use of them can cause problems to web sites. When you use bots you always must understand what you're doing and, for example, you must always delay calls.

For example if you are going to use the getUrlInfo method to get informations on many URLs remember that each time you call getUrlInfo you are doing web calls to a server. So, if you make 1.000 calls fast inside a for loop (for example) you can cause a crash to the target server, or the target server could have filters to block and ban the server from which you're making calls. If you are on a shared server you can cause problems also to the other sites on the same server.
So, before run your code with this bots, think to what bots are doing.

Moreover: some of the methods of this class uses free APIs provided by the owner of the services, for example the Vimeo api to retrieve infos about a video, or Twitter APIs to retrieve infos about tweets, or Facebook graph APIs. Some other APIs used by these bots have usage limits (as one used in ipToGeo method) and I've reported links in this documentation to use them without policies problems.

You can find a working demo with many of these webbots here.

I sometimes post tutorials and updates about this class on my blog Barattalo.it.

B) Be careful and use cache - top

As I told you in the previous chapter, Bots are small programs that can cause problems and can also be used badly, for example to copy data. Since this data are public, it should not be a problem (counters of facebook, or meta description of pages that are made just for spiders, or your images on Instagram) but sometimes public data may have copyrights, so be careful before making your spiders and using this class.

A big good suggestion is cache all the data. These bots may not use APIs and may work directly with the target site, for example, such as reading images from Instagram. Althought Instagram servers are very powerful you should always save locally the data you grab with bots, because you can cause problems to server and you can slow down your software. So, please, use cache everytime you can.

C) Class parameters - top

At the top of this class definition there are some parameters.
The use_file_get_contents parameter can be these values:
yes = to force the getPage method used in many methods of this class to use the php function file_get_content to retrieve data.
no = to force the getPage method used in many methods of this class to use CURL to retrieve data. https = to force the getPage method used in many methods of this class to use file_get_content for https calls and CURL to http calls. If some methods doesn't work, try to change this variable, for example:

<?php
$mb->use_file_get_contents="yes"; // this works with twitter method for my hosting
$data = $mb->twitterInfo("dailybestnet");
print_r($data);
?>

D) Methods

getIP() - top

The getIP method gets the IP address of the connected user:
Usage:
input params: none
output: string with IP address

<?php
$mb = new Minibots();
$ip = $mb->getIP();
// example: 93.36.140.211
?>

copyFile($url,$filename) - top

The copyFile method save a remote url as a local file on your server:
Usage:
input params: $url is the remote address url, $filename is the local file to save
output: true if success or false if fails

<?php
$mb = new Minibots();
if($mb->copyFile("http://1.s3.envato.com/files/68503809/pons-80x80.jpeg","avatar.jpg")) {
	echo "ok";
} else {
	echo "fails";
}
?>
Note: remember that you must have write permissions to save files locally.

doSpelling($q) - top

The doSpelling method search Google to find spelling suggestion:
Usage:
input params: $q is the mispelled string to check
output: if found something returns corrected string else returns input $q string

<?php
$mb = new Minibots();
$s = $mb->doSpelling("wikipezia");
echo $s;
?>
Example output: 
wikipedia
Note: this method doesn't use an API provided by Google and should be used carefully, it queries Google search engine directly and parse html to find the correct spelling.

doShortURL($longUrl) - top

The doShortURL method uses tiny url api to make a short url from a long one:
Usage:
input params: $longUrl is the long URL you want to shorter
output: shortened url

<?php
$mb = new Minibots();
$s = $mb->doShortURL("http://www.dailybest.it/2013/03/05/vita-programmatore-gif-animate/");
echo $s;
?>
Example output: 
http://tinyurl.com/p5u8fjz
Note: this method uses Tiny Url services, free of use.

doShortURLDecode($url) - top

The doShortURLDecode method converts back a small url to the original one. Since this function decodes redirects it works with all url shortener services such as goo.gl, tinyurl.com, bit.ly...
Usage:
input params: $url is the short URL you want to decode
output: original url

<?php
$mb = new Minibots();
$s = $mb->doShortURLDecode("http://tinyurl.com/p5u8fjz");
echo $s;
?>
Example output: 
http://www.dailybest.it/2013/03/05/vita-programmatore-gif-animate/

checkMp3($url) - top

The checkMp3 method uses cURL to check if the file specified in the URL is an mp3 file with the audio/mpeg mime type. It uses only headers and doesn't download the mp3 file.
Usage:
input params: $url of the remote mp3 file
output: true / false

<?php
$mb = new Minibots();
if($mb->checkMp3("http://www.artintent.it/Kalimba.mp3")) {
	echo "yes";
} else {
	echo "no";
}
?>
Example output: 
yes

url_exists($url) - top

The url_exists method uses cURL to check if calling the specified URL returns a 200 OK response or not. This can be used to check if links exist.
Usage:
input params: $url of the remote link to test
output: true / false

<?php
$mb = new Minibots();
if($mb->url_exists("http://en.wikipedia.org/wiki/Barack_Obamaxxxxxxxxx")) {
	echo "yes";
} else {
	echo "no";
}
?>
Example output: 
no

doSMTPValidation($email, $from_address="", $debug=false) - top

The doSMTPValidation method performs an SMTP check of an email address by connecting to the mail server of the destination address and asking for the specified mailbox.
Usage:
input params: $email is the address to check, $from_address is the address of the sender, $debug is useful to understand how mail servers work.
output: if email is ok returns true, else returns an array with two values, the first is the $error string from mail server and the second is the error number $code.

$mb = new Minibots();
$res = $mb->doSMTPValidation("pons@rockit.it","noreply@barattalo.it",true);
if ($res===true) echo "ok";

?>

getPage($url,$max_file_size=0) - top

The getPage method is a facility for making a cURL call and retrieve a page.
Usage:
input params: $url is the target page to retrieve, $max_file_size it's a limit to file size when grabbing pages, to avoid downloading big pages, it could be used if you have to grab a lot of pages, by default it's set to 0, that means OFF.
output: the html of the target page

Note: grabbing HTTPS urls with cURLs is tricky and sometimes it doesn't work. For this reason if it doesn't work the getPage method can be configured to work with file_get_contents which sometimes works also with https calls. If you want to use file_get_contents instead of cURLs, set the variable $use_file_get_contents_https to false at the beginning of the class (near line 20).
